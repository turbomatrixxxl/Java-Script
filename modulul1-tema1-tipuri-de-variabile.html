Introducere în limbajul Javascript

Când vorbim despre programare, prima dată ne vine în minte un set de instrucțiuni dintr-un fișier. Codul sursă - este textul unui program pentru orice limbaj de programare, un set de construcții sintactice lizibile care descriu un set de instrucțiuni pentru un calculator.

Calculatorul nu înțelege codul sursă, din acest motiv există un pas adițional care convertește codul dintr-un fișier într-un set de instrucțiuni clare. Această funcție îi revine unui program special numit: compilator sau interpretor. Deci, codul nu e scris pentru dispozitiv (calculator), ci pentru programator. Codul sursă trebuie să fie scris corect pentru a îndeplini un task bine definit, cât și clar și ușor de înțeles.

Gândirea logică

Există o mulțime de limbaje de programare, însă niciunul nu-i atât de dificil ca și cel uman, deoarece sunt compuse dintr-un set mai mic de instrucțiuni sintactice, iar principiul de funcționare și conceptele de bază sunt asemănătoare în toate limbajele de programare.

Programatorii experimentați privesc o problemă din punct de vedere algoritmic, adică e alcătuită din pași elementari necesari soluționării acesteia. Noi folosim zilnic algoritmii in viata de zi cu zi. De exemplu, procesul de pregătire a unui ceai - este un set alcătuit din mai mulți pași simpli. Când veți fi obișnuiți să gândiți algoritmic cu privire la soluția unei probleme, orice limbaj de programare va fi doar un tool pentru a ajunge la soluția dorită.

JavaScript

JavaScript este o implementare a specificației EcmaScript, un limbaj de programare de nivel înalt, suportat de toate browserele web moderne. A fost proiectat inițial pentru a interacționa cu elementele paginilor web și pentru un plus de interactivitate.

BINE DE ȘTIUT
JavaScript nu are nici o legătură cu limbajul Java. Acestea sunt 2 limbaje independente și total diferite.

Pentru dezvoltarea de Front-end a unei aplicații web, JavaScript se folosește împreună cu HTML și CSS pentru a asigura funcționalitatea unei pagini web, cum ar fi:

  • calcule simple
  • validarea și manipularea datelor introduse de utilizator
  • stocarea informațiilor în browser
  • modificare dinamică a documentului HTML
  • reacția la acțiunile utilizatorului
  • crearea de elemente interactive: galerii, grafice etc.
  • interacțiunea cu backend-ul

BINE DE ȘTIUT
Dacă codul JavaScript este scris pentru a fi executat în browser (Front-end), programatorul nu are instrumente și acces la fișierul sau sistemul de operare, din motive de securitate.

La ora actuală, cu ajutorul limbajului JavaScript putem construi:

  • aplicații web, folosind framework-uri ca React, Vue, Angular etc.
  • Back-end/servere cu Node.js
  • aplicații mobile, folosind React Native sau Ionic
  • aplicații desktop, folosind Electron
  • microcontroalere cu Johnny-Five și Espruino

Pentru realizarea acestora avem nevoie de cunostințe solide a sintaxei, specificațiilor și mecanismului intern și de a antrena gândirea algoritmică și rezolvarea problemelor, în număr cât mai mare.

Adăugarea unui script

Pentru a adăuga un script într-o pagină web, în fișierul HTML adăugăm tag-ul script, iar la atributul src menționăm calea externă către fișierul JavaScript.

Pentru a adăuga cod JavaScript dintr-un fișier extern:

  • Creați un fișier cu extensia .js si adăugați-l in subfolderul js.
  • Menționați calea de acces către fișierul creat în atributul src din tag-ul script.

BINE DE ȘTIUT
Fișierul cu cod JavaScript nu este necesar să se afle într-un folder numit js, însă este o practică bună.

Dacă script-ul este adăugat în (head), atunci randarea paginii se oprește până în momentul în care acel script este încărcat și executat complet. Browser-ul încarcă și reflectă HTML-ul progresiv. Dacă întâlnește tag-ul (script) fără atribute suplimentare, atunci se execută script-ul și doar după aceea se prelucrează codul HTML rămas. Din acest motiv se adaugă script-ul înainte de a închide tag-ul (body).

Atașarea mai multor script-uri

La conectarea unor fișiere cu cod JavaScript, interpretorul prelucrează script-urile în acea ordine în care sunt atașate în fișierul HTML.

Tool-uri necesare

În timp ce scrieți cod, vor apărea avertismente și erori. Consola unui browser afișează informații legate de pagina web, inclusiv mesaje despre erori și avertismente de execuție a codului JS care rulează în contextul paginii.

Următoarele combinații de taste vor deschide instrumentele folosite de un programator în tab-ul Console:

  • Windows și Linux - Ctrl + Shift + J sau F12
  • MacOS - Command + Option + J

Resurse adiționale

  • Printarea unui mesaj în consolă
  • Handbook despre API console

Sintaxa de bază

Când scrieți cod, nu este important să știți doar ce simbol sau construcție pot fi folosite, dar, în primul rând, este necesar să înțelegeți terminologia și componentele codului sursă. În această secțiune, nu este important pentru noi să înțelegem cum și ce funcționează, ci ne vom familiariza doar cu terminologia și sintaxa de bază.

Instrucțiuni de control

O instrucțiune de control este un set de cuvinte și simboluri din sintaxa specifică unui limbaj care împreună exprimă o idee/o instrucțiune pentru un calculator.

a = b * 2;

Instrucțiunile JavaScript se termină cu punct și virgulă, acest aspect fiind similar cu ceea ce reprezintă punctul la sfârșitul unei propoziții.

  • a și b - variabile (ca într-o ecuație algebrică) care stochează date ce pot fi folosite, ulterior, într-un program. O variabilă e compusă dintr-un identificator (nume) și o valoare asociată acestuia.
  • 2 - un număr. Acesta se numește "literal value", deoarece nu este stocată într-o variabilă.
  • = și * - operatori ce efectuează acțiuni asupra valorilor și a variabilelor.

Să ne imaginăm că variabila b stochează deja numărul 10. Apoi această instrucțiune îi spune calculatorului:

  1. Caută variabila cu id b și află ce valoare are stocată.
  2. Înlocuiește în enunț valoarea variabilei b (10) cu b.
  3. Efectuează operația de înmulțire la 10 cu 2.
  4. Scrie rezultatul evaluării expresiei, în partea dreaptă a variabilei a.

BINE DE ȘTIUT
Încheierea unei instrucțiuni cu punct și virgulă nu este obligatorie, dar este recomandat să o puneți în practică. Această regulă simplă va face codul mai ușor de înțeles și va ajuta la evitarea unor erori.

Expresiile

Instrucțiunile sunt alcătuite din mai multe părți, la fel ca în orice limbă, frazele sunt alcătuite din propoziții, iar aceste propoziții sunt numite expresii.

Expression - fac referire la o variabilă, o valoare sau la un set de variabile și valori în combinație cu operatorii aritmetici.

[ [a] = [ [b] * [2] ] ]

Instrucțiunea din exemplul de mai sus conține 5 expresii și sunt încadrate în paranteze pătrate pentru a fi mai ușor vizualizate (nu este sintaxa limbajului):

  • [2] - expresia valorii literale.
  • [b] și [a] - sunt expresiile variabilelor; acestea necesită înlocuirea valorii unei anumite variabile, dar numai dacă aceasta se află în partea dreaptă a expresiei de atribuire.
  • [b * 2] - expresie aritmetică pentru înmulțire.
  • [a = b * 2] - expresie de atribuire. În cazul nostru, indică necesitatea de a evalua partea dreaptă a expresiei și de a atribui rezultatul variabilei a din partea stângă a expresiei.

Există și expresii de apel, comparații etc. Nu le vom lua în considerare acum pe toate, ci este important pentru noi să înțelegem din ce părți este format codul sursă și cum să-l citim corect.

Interface

Când ne apropiem de un aparat de cafea sau ne urcăm la volanul unei mașini, există un set de comenzi cu care putem interacționa. În programare se numește "interface".

Interface - este un set de proprietăți și metode ale unei entități disponibile pentru utilizare în codul sursă.

Proprietăți

Noi toți avem proprietăți: înălțime, greutate, culoarea ochilor, adică niște caracteristici descriptive. În mod similar, datele au proprietăți, de exemplu, un șir are o proprietate pentru lungimea sa. Sintaxa pentru accesarea unei proprietăți este foarte simplă - printr-un punct.

entitate.numele_proprietății

Pentru claritate, să ne uităm la proprietatea unui șir - length, conține numărul de caractere din șir.

"JavaScript is awesome".length;

Metode

O metodă este un apel la o acțiune, cum ar fi aleargă sau scrie. De asemenea, datele au propriile metode predefinite, de exemplu, putem adăuga sau elimina elemente dintr-o colecție, putem converti un șir etc. Sintaxa pentru apelarea unei metode este foarte asemănătoare cu accesarea unei proprietăți, dar o pereche de paranteze se adaugă la sfârșit.

entitate.numele_metodei()

Ca exemplu, să apelăm metoda toUpperCase(), tehnică ce va face ca toate literele dintr-un string să fie scrise cu majuscule.

"JavaScript is awesome".toUpperCase();

Strict mode

O nouă caracteristică în ECMAScript 5 permite să punem script-ul în deplină conformitate cu standardul modern. Acest lucru previne anumite erori, cum ar fi utilizarea unei sintaxe nesigure și depreciate. Pentru a pune script-ul în strict mode, este suficient să specificăm directiva, la începutul fișierului js. Scrieți întotdeauna codul în strict mode.

"use strict";
// Este un comentariu doar. În continuare va fi întregul cod din fișierul JS

Variabile și tipul variabilelor

Variabilele sunt folosite pentru stocarea unor date și sunt formate dintr-un identificator (nume) și o zonă alocată în memorie în care este stocată valoarea. O variabilă poate fi gândită ca o cutie ce are un nume și conține ceva (valoare).
